Keeping this site secure — admin password & session secret

For safety, the admin password and session secret are not editable here. They are stored as encrypted deployment secrets, so they can never leak through this dashboard or the database.

Where to set or change them: in the Cloudflare dashboard go to Workers & Pages → your Worker → Settings → Variables and Secrets, edit ADMIN_PASSWORD or SESSION_SECRET, then Save and deploy. (Technical users can instead run wrangler secret put ADMIN_PASSWORD.)

  • Admin password — use a long, unique passphrase from a password manager. Change it if anyone who shouldn’t have it may have seen it, when someone with access leaves, and every few months as good hygiene.
  • Session secret — a long random value (e.g. generated with openssl rand -base64 32). Set it once and leave it. Change it only if you suspect sign-ins were compromised — doing so signs out all admins, who then simply log in again.
  • Never put either value in code, screenshots, or chat messages.

Step-by-step instructions are in the project README under “Security”.

Import members from CSV

Upload a CSV exported from this tool, or your raw sign-up sheet. Column headers are detected automatically and each location is geocoded to a map pin you can review before importing. Rows that don’t resolve can still be imported as pending drafts — copy their edit link afterwards so the member can set their own location.

What should the CSV look like?

A header row plus one row per member. Headers are matched flexibly (case-insensitive), so your sign-up sheet’s wording usually works as-is. Recognised columns:

  • Name (required) — the member’s display name.
  • Location — city / place; geocoded to a pin. You can instead provide Latitude & Longitude columns to skip geocoding.
  • Contact — a link, email or @handle.
  • Email — kept private (never shown on the map).
  • Bio — a short description.
  • Status — published, pending or hidden (defaults to published).
  • Consent — Yes/No for public visibility (defaults to Yes).
  • Image — a filename (e.g. jane.jpg) when importing photos; see below.

Tip: export the current directory first to see the exact column layout this tool produces.

Importing photos (optional): tick the photos box below and attach a .zip of images. Add an Image column to your CSV whose value is each member’s filename inside the zip (e.g. jane.jpg, or images/jane.jpg — folders are ignored). Photos are automatically resized and compressed on import.
